# 使用者快速入門 — 用 Claude 查系統邏輯

> 對象:一般使用者(不需工程背景),使用 Claude Desktop。
> 接上團隊的「系統知識庫」後,可以直接問 Claude:「這個功能的規則是什麼?」
> 答案來自**現在正在跑的系統**,不是過期文件,並附依據。

## 安裝(一次,約 3 分鐘)

先向團隊管理員拿**連線網址(Connector URL)**。
(公司若已由管理員統一發佈,打開 Claude Desktop 就有現成連接器,直接跳到「開始提問」。)

1. Claude Desktop → 左下角名字 → **Settings** → **Connectors**
2. **Add custom connector** → 名稱隨意 → 貼上網址 → 確認
3. 回到聊天,點輸入框旁的工具圖示,確認連接器是開啟狀態

找不到 Connectors 選單 = 方案不支援或需管理員開權限,請聯絡管理員。

## 開始提問

用平常說話的方式問,**講出系統名稱**(團隊管很多系統):

- 「**報價系統**的總金額是怎麼算出來的?」
- 「**請假系統**的加班上限是多少?可以改嗎?」
- 「**訂單系統**篩選異常訂單的規則是寫死的還是設定的?」

不知道系統叫什麼,先問:「你可以查哪些系統?」任何語言都可以問。

## 怎麼判斷答案可信

回答會附依據:

| 依據 | 意思 |
|---|---|
| `HouseService.java:42` | 邏輯寫在這個檔案這一行 |
| 「DB 現值:0.40」 | 資料庫**現在**存的設定值 |
| config key | 系統設定檔的參數 |

**沒附依據的段落當推測看待**;要拿去做決策的數字,確認有標「DB 現值」。

## 常見問題

- **查不到 / 答非所問**:換個說法(用畫面上的功能名稱);還是不行就把問法
  回報管理員,通常是對照表缺詞,補一條就好。
- **答案會過期嗎**:程式邏輯索引每天自動更新(最多差幾分鐘~幾小時);
  DB 設定值與系統設定檔是即時查的。
- **會不會改壞系統**:不會,這個服務只能讀不能寫;個資、密碼查不到(密碼顯示遮罩)。
- **適合問什麼**:規則、公式、設定、流程、影響範圍。它不能操作系統、改資料,
  也看不到你的個別資料。
- **連線錯誤**:確認在公司內網(或 VPN),再找管理員確認服務狀態。
